# BackTest version3

무한매수법 V4와 전고점매매법을 같은 데이터·원금·비용 조건에서 실행하고 비교하는 독립 백테스터입니다. 기존 무한매수법 V4 엔진과 결과 화면은 그대로 유지하면서 version 3.0에 SOXX/SOXL 전고점매매법, 6전략 비교, 6전략 랜덤 기간 비교, 파라미터 안정성 분석을 추가했습니다. 이 저장소는 LTH 웹앱과 별도로 설치·실행·배포할 수 있습니다.

## 가장 쉬운 실행

프로젝트 루트에서 다음 명령을 실행합니다. 외부 패키지 설치는 필요하지 않습니다.

```bash
./run
```

다른 포트를 쓰려면 `./run --port 9000`처럼 실행 옵션을 뒤에 붙입니다. `python3 -m lth_backtest.cli serve --open` 명령도 계속 사용할 수 있습니다.

브라우저가 자동으로 열리지 않으면 터미널에 표시된 `http://127.0.0.1:8765`로 접속합니다.

브라우저에서는 먼저 분석 모드를 고릅니다.

1. `무한매수 V4`: 기존 TQQQ/SOXL 무한매수법을 단독 분석
2. `전고점`: SOXX를 기본 보유하고 하락 단계마다 SOXL로 전환하는 전략을 단독 분석
3. `6전략 비교`: 동일 기간과 비용으로 무한매수법 V4, 전고점매매법, SOXL·SOXX·TQQQ·QLD Buy & Hold를 비교

공통 원금·기간·수수료·슬리피지는 한 번만 입력하며, 선택한 분석 모드에 필요한 전략 설정과 데이터 입력만 표시됩니다. 전고점 단독 모드는 SOXX·SOXL, 6전략 비교 모드는 SOXX·SOXL·TQQQ·QLD에 모두 존재하는 거래일만 사용합니다.

브라우저 UI에서 할 수 있는 작업:

- 무한매수법 V4의 TQQQ/SOXL, 20/40분할과 기존 호환용 실험적 30분할
- 전고점매매법의 매매 간격, 분할 수, 정수/소수점 주식, 청산 오프셋 설정
- 전체 가격 데이터의 실제 거래일을 사용하는 시작일·종료일 양방향 슬라이더와 랜덤 기간 설정
- 장중 고가 기준과 종가 전용 기준의 결과 자동 비교
- 수수료, 매도 비용, 슬리피지, 무위험 수익률 설정
- 전고점 단독 대시보드의 전고점매매법/SOXX 거치식/SOXL 거치식 3곡선과 비교 실행 시 V4/전고점/SOXL/SOXX/TQQQ/QLD 6곡선·낙폭 확인
- 선택 기간의 실제 OHLC 캔들·거래량 차트, 국내식 상승 빨강·하락 파랑 표시와 60/120/252거래일 구간 이동
- CAGR, MDD, 변동성, Sharpe, Sortino, Calmar, 노출도, 회전율
- 모든 체결의 주문가·체결가·현금·수량·T값 추적
- 라운드별 전략·종목 거치식 수익률과 종가 MDD·월별·연도별 결과, 6전략 랜덤 기간 견고성 분석
- 설정 기간의 모든 거래일을 독립 시작점으로 한 1라운드 전수 분석
- 전고점매매법의 SOXL 비중, 실질 레버리지, SOXX 낙폭별 포트폴리오 구조, 라운드·전환 로그
- 매매 간격 × 분할 수 CAGR·MDD·Calmar 히트맵과 전체/초기·중간·최근 1/3 안정성 분석
- JSON, 체결 CSV, 단독 실행형 HTML 리포트 내보내기
- Yahoo 실제 OHLCV 4종목 다운로드(분할 반영·배당 미보정)

브라우저 왼쪽 설정 패널 맨 아래의 `전체 가격 데이터` 영역에서 `전체 데이터 갱신`을 누르면 TQQQ, SOXX, SOXL, QLD 네 종목의 전체 가용 이력을 오늘 날짜까지 한 번에 갱신합니다. 이 작업은 백테스트 분석 기간 설정과 독립적입니다. 2.4.0 이전에 받은 데이터는 배당까지 소급 조정된 가격이므로 업그레이드 후 한 번 다시 갱신해야 합니다. version 3.0에서 전고점매매법을 처음 사용하는 경우에도 SOXX 파일을 받기 위해 한 번 갱신해야 합니다.

분석 기간의 양쪽 슬라이더는 선택한 종목 CSV의 전체 거래일을 기준으로 움직입니다. 주말·휴장일은 건너뛰며 시작일·종료일 입력란과 양방향으로 동기화됩니다. 데이터 목록에 없는 사용자 CSV 경로를 직접 입력하면 슬라이더 대신 날짜 입력란을 사용합니다.

서버 종료는 실행한 터미널에서 `Ctrl+C`입니다.

### 캔들 차트

일반 백테스트를 실행한 뒤 `캔들 차트` 탭을 열면 설정한 기간의 실제 OHLC와 거래량을 확인할 수 있습니다. 가격은 주식분할을 반영하지만 배당은 과거 가격에서 소급 차감하지 않습니다. 굵은 몸통은 시가와 종가, 얇은 꼬리는 고가와 저가를 나타내며 상승봉은 빨강, 하락봉은 파랑입니다. 기본값은 마지막 120거래일이며 60일, 252일, 전체 기간으로 바꾸거나 이전·다음 구간을 이동할 수 있습니다. 캔들 위에 포인터를 올리면 날짜, 시가, 고가, 저가, 종가, 시가·전일 대비 등락률, 거래량을 표시합니다.

### 전고점매매법

기본값은 `매매 간격 5% × 20분할`, 정수 주식, 전고점 청산 오프셋 0%, 거래비용 0입니다. 시작일 종가에 원금으로 SOXX를 매수하고 그 종가를 최초 전고점으로 기록합니다. 기준금액은 전고점 기록 시점의 전체 포트폴리오 평가액을 분할 수로 나눈 값이며, 다음 전고점이 기록될 때까지 현재 평가액과 관계없이 유지됩니다.

전고점에서 5%, 10%, 15%처럼 설정 간격만큼 내려갈 때마다 아직 실행하지 않은 단계 수에 해당하는 기준금액을 SOXX에서 SOXL로 전환합니다. 한 라운드에서 실행한 단계는 반등 후 다시 내려와도 중복 실행하지 않습니다. SOXX가 부족하면 남은 수량을 모두 전환하고, SOXX가 처음 소진된 당시의 전고점 대비 낙폭을 결과에 기록합니다.

일봉 한 개는 다음 순서로 처리합니다.

1. 시가에서 전고점 회복 여부 확인
2. 회복하지 않았다면 시가에서 신규 하락 단계 전환
3. 종가에서 전고점 회복 여부 확인
4. 회복하지 않았다면 종가에서 추가 신규 하락 단계 전환
5. SOXL이 없는 상태에서 종가가 기존 전고점보다 높으면 전고점·기준금액 갱신

SOXL 보유 중에는 회복 청산을 하락 전환보다 먼저 검사합니다. 시가가 조건을 충족하면 SOXX와 SOXL 모두 시가로, 종가가 조건을 충족하면 모두 종가로 체결합니다. 같은 날 시가에서 여러 단계를 건너뛰면 해당 단계의 기준금액을 합산해 한 번에 전환하고, 종가에서 더 깊은 신규 단계가 생기면 종가에서 추가 전환합니다. 장중 High/Low만 조건을 터치하고 시가와 종가가 충족하지 않으면 아무 거래도 하지 않습니다.

SOXX→SOXL 하락 전환은 매도·매수 두 주문이 모두 가능한 경우에만 원자적으로 실행합니다. 회복 시에는 SOXL을 전량 매도하고 그 순대금만으로 정수 단위 SOXX를 매수하며, 기존 잔여 현금을 매수 재원에 섞지 않습니다. 순대금이 SOXX 1주보다 작으면 매수 수량은 0주이고 순대금은 현금으로 남습니다. 슬리피지는 매수에 가산하고 매도에서 차감합니다. 고정 수수료는 실제 발생한 각 주문에 적용하며, 매도 비용 bp는 매도 총액에 추가합니다.

기본 정수 모드는 매매 가능 수량을 주 단위로 버리고 잔액을 현금으로 유지합니다. `소수점 주식`을 켜면 수량을 소수 8자리에서 버리는 이론적 체결을 사용합니다. 실제 증권사의 지원 자릿수 및 체결 방식과 다를 수 있습니다.

전고점 전용 결과에는 다음이 포함됩니다.

- 라운드 시작·첫 전환·종료일, 회복 거래일, 시작 전고점과 기준금액
- 라운드 수익률, 시가·종가 판정시점 MDD, 최대 SOXX 낙폭, 최대 SOXL 비중, 최대 실질 레버리지
- 전환 단계 수·전환 이벤트 수·주문 수·총 비용, SOXX 소진 여부와 소진 낙폭
- 전체 평균·중앙·최장 라운드 기간, 평균·최악 라운드 수익률
- 일별 SOXX/SOXL/현금 비중과 `SOXX 비중 × 1 + SOXL 비중 × 3` 실질 레버리지
- SOXX 낙폭 구간별 평균 비중·레버리지·포트폴리오 손익

### 6전략 비교와 파라미터 안정성

`6전략 비교`는 SOXX·SOXL·TQQQ·QLD의 공통 거래일, 같은 초기자본, 진입 슬리피지와 고정 수수료를 사용하여 다음 여섯 비교선을 계산합니다. 소수점 수량 옵션은 전고점매매법과 Buy & Hold 비교선에 적용되며, 무한매수법 V4는 기존 엔진의 정수 주식 규칙을 그대로 유지합니다.

- 무한매수법 V4.0
- 전고점매매법
- SOXL Buy & Hold
- SOXX Buy & Hold
- TQQQ Buy & Hold
- QLD Buy & Hold

최종 자산, 총수익률, CAGR, 종가 MDD, Calmar, 변동성, Sharpe, Sortino, 연도별 수익률과 초기·중간·최근 1/3 및 구현된 주요 시장 구간을 함께 확인합니다. Buy & Hold도 첫날 종가에 실제 수량을 계산하고 남는 금액을 현금으로 유지합니다. 비교 탭의 가설 카드는 관찰된 차이를 요약하며 특정 매매 구조의 인과 효과를 증명하는 통계 검정은 아닙니다.

`파라미터 탐색`은 기본 후보인 간격 `2.5, 3, 4, 5, 6, 7.5, 10%`와 분할 수 `10, 15, 20, 25, 30, 40`의 조합을 계산합니다. 각 조합은 전체 기간뿐 아니라 초기·중간·최근 1/3을 동일 원금으로 독립 실행합니다. 안정성 점수는 각 구간의 CAGR·Calmar 순위와 바로 이웃한 간격/분할 조합의 성과 변화폭을 함께 반영합니다. 따라서 가장 높은 CAGR 한 개를 자동 최적값으로 확정하지 말고, 주변 조합에서도 결과가 유지되는 넓은 구간을 확인해야 합니다.

비교 모드의 `랜덤 비교` 탭은 왼쪽 패널에서 선택한 무한매수법 V4의 분할·운용 설정과 전고점매매법의 간격·분할 설정을 고정합니다. 무작위로 뽑은 모든 기간에 여섯 전략을 똑같이 적용하고, 전략별 평균·중앙·최악·최고 수익률, 평균·최악 MDD, 수익률 1위 비율, 최소 MDD 비율, 평균 수익 순위를 카드와 구간별 상세표로 제공합니다. 모든 샘플의 수익률·MDD P10/P25/중앙/P75/P90 분포와 1~6위 진입 확률도 전체 폭 차트로 비교합니다. 최대 5,000개 샘플을 백그라운드에서 계산하며 완료 수·진행률·경과시간·예상 남은 시간을 화면에 표시합니다. 재현 시드를 입력하면 동일한 랜덤 기간과 결과를 다시 얻을 수 있습니다.

무한매수 V4 모드의 `랜덤 비교`도 선택한 종목 × 분할 수 조합을 전체 폭으로 비교합니다. 조합별 기존 평균 성과와 거치식·QLD 대비 결과를 유지하면서 수익률·MDD 분포, 같은 구간 내 수익 순위 확률, 구간별 상세표를 함께 제공합니다. 선택 종목과 QLD의 공통 거래일만 사용하므로 모든 조합은 정확히 같은 거래일 구간에서 비교됩니다.

`균등 시작일 샘플링`을 선택하면 기간 길이와 시작일을 무작위로 뽑지 않습니다. 모든 표본 길이를 `최소 거래일`로 고정하고, 그 길이를 끝까지 확보할 수 있는 유효 시작일 전체에 요청 표본을 일정 간격으로 배치합니다. 이 모드에서는 최대 거래일과 재현 시드를 사용하지 않습니다. 요청 샘플 수가 유효 시작일 수보다 많으면 가능한 시작일 수까지만 자동 적용하고 요청 수와 실제 적용 수를 결과에 함께 표시합니다.

### 시작일별 1라운드 분석

브라우저의 `시작일별 1라운드` 탭에서 `모든 시작일 분석 실행`을 누르면 설정 기간에 존재하는 각 거래일을 첫 매수일로 삼아 독립적인 1라운드를 계산합니다. 첫 전량매도가 나온 즉시 해당 표본을 종료하며 다음 정보를 표, 요약 카드, 시작일별 수익률·MDD 흐름과 위험 대비 수익 분포 차트로 제공합니다.

- 시작일, 전량매도 종료일, 달력일·거래일 수
- 완료 손익·수익률, 일별 종가 평가자산 기준 MDD
- 라운드 중 최대 T값과 종료 시점 T값
- 리버스 진입·복귀 횟수와 종료 모드
- 매수·매도·전체 체결 횟수, 최대·종료 보유수량, 비용
- 완료율, 완료 표본 평균·중앙 수익률, 평균 기간, 리버스 비율 등 전체 요약

각 표본은 시작일 종가에 첫 1회분을 MOC 매수합니다. 설정 종료일까지 전량매도가 나오지 않은 표본은 `미종료`로 표시하고 종료일 종가로 평가하되, 완료 라운드 수익률·기간 평균에서는 제외합니다. 모든 행은 JSON 또는 CSV로 내려받을 수 있습니다.

  --html-out results/tqqq.html \

다운로드부터 실행까지:

python3 -m lth_backtest.cli previous-high \

`previous-high-sweep`는 `ph-sweep` 별칭도 지원합니다. 전고점 단독·탐색은 기본 `data/SOXX.csv`, `data/SOXL.csv`를 사용합니다. 6전략 비교는 여기에 `data/TQQQ.csv`, `data/QLD.csv`를 더 사용하며 `--soxx-csv`, `--soxl-csv`, `--tqqq-csv`, `--qld-csv`로 다른 파일을 지정할 수 있습니다.

## 가격 데이터

필수 CSV 열:

```text
date,open,high,low,close,adj_close,volume,price_basis
```

- 소문자 헤더와 Yahoo 형식의 `Date`, `Adj Close`를 모두 읽습니다.
- 전략 체결과 캔들에는 Yahoo 일반 `Open/High/Low/Close`를 그대로 사용합니다. 이 가격은 주식분할은 반영하지만 배당은 소급 반영하지 않습니다.
- `Adj Close`는 데이터 검증용으로 별도 보관하며, 그 비율을 OHLC에 곱하지 않습니다.
- 내려받은 파일에는 `price_basis=actual_split_adjusted`를 기록합니다. 이 표식이 없는 기존 관리 데이터는 잘못 섞어 쓰지 않도록 전체 갱신을 요구합니다.
- ETF 현금 분배금은 전략 현금에 따로 가산하지 않습니다. 실제 주문가격 재현을 우선한 가격수익률 백테스트이므로 배당 재투자 총수익률과는 다를 수 있습니다.
- 중복 날짜, 0 이하 가격, `high < open/close/low`, `low > open/close/high`는 즉시 거부합니다.
- 기본 데이터와 새로 다운로드한 데이터는 저장소의 `data` 디렉터리에 저장합니다.
- 사용자 CSV는 브라우저의 가격 CSV 입력란에 경로를 직접 입력할 수 있습니다. `price_basis`가 없으면 `user_provided`로 기록하며, 엔진·캔들·HTML 리포트는 분할·배당 조정 여부를 확인할 수 없는 입력을 실제 거래가격이라고 표시하지 않습니다. 열이 존재한다면 모든 행에 동일한 값을 넣어야 합니다.
- 전고점매매법 단독 실행은 SOXX와 SOXL, 6전략 비교와 랜덤 비교는 SOXX·SOXL·TQQQ·QLD에 같은 날짜가 실제로 존재하는 행만 사용합니다. 한 종목의 누락일을 다른 종목의 전일 가격으로 채우지 않으며, 제외된 날짜 수와 예시를 진단 결과에 남깁니다.

SOXL은 2010년에 상장했으므로 실제 SOXL을 사용하는 전고점매매법과 6전략 비교의 시작일은 네 종목 공통 데이터 시작일보다 앞설 수 없습니다. 현재 구현은 SOXL 상장 전 가격을 합성하거나 실제 SOXL 이력에 이어 붙이지 않습니다. 따라서 닷컴버블과 2008 금융위기는 현재 실제 6전략 결과로 분석할 수 없으며, 3배 일일 수익률 합성 모델을 별도 구현하기 전까지 임의의 과거 SOXL 결과로 해석하면 안 됩니다.

## 정확한 체결 규칙

| 주문 | 체결 조건 | 기본 체결가 |
|---|---|---|
| LOC 매수 | `당일 종가 <= 주문가` | 당일 종가 |
| LOC 매도 | `당일 종가 >= 주문가` | 당일 종가 |
| 최종 LIMIT 매도 | `당일 고가 >= 목표가` | 목표 지정가 |
| MOC | 거래일 존재 | 당일 종가 |

최종 LIMIT 주문과 LOC 주문은 장 시작 전에 동시에 제출된 것으로 봅니다. LIMIT 매도는 장중 고가로 먼저 판정하고, LOC는 종가로 판정합니다. 따라서 고가가 최종 목표가를 넘은 뒤 종가가 내려오면 다음이 같은 날 함께 발생할 수 있습니다.

1. 시작 보유량 중 최종 지정가에 예약한 수량 매도
2. 쿼터 LOC 매도는 종가가 별지점 이상일 때만 체결
3. 수량이 남으면 종가가 조건을 만족한 LOC 매수 체결

이 경우 T값도 문서의 `직전 T × 0.25 + 매수 효과` 순서로 계산합니다.

더 자세한 전략 대응표와 한계는 [METHODOLOGY.md](./METHODOLOGY.md)에 있습니다.

## 중요한 한계

- Yahoo 일봉 고가는 보통 정규장 OHLC입니다. `trade_original.md`가 의도한 프리장·애프터장까지 포함하려면 해당 세션을 합친 고가가 들어 있는 사용자 CSV가 필요합니다.
- 일봉만으로는 고가와 저가가 발생한 순서, 호가 스프레드, 거래정지, 부분체결, 주문 거절을 재구성할 수 없습니다.
- LOC는 공식 종가에 전량 체결된다고 가정합니다. 실제 증권사별 LOC 처리와 주문 제출 시각은 다를 수 있습니다.
- 전고점매매법은 검증 가능한 시가와 종가만 신호로 사용하므로 장중 High/Low 터치는 의도적으로 무시합니다. 일봉만으로 시가 이후와 종가 이전의 경로를 복원하지 않습니다.
- SOXX와 SOXL을 같은 시점 가격으로 전환할 수 있다고 가정하며 실제 호가 스프레드, 부분체결 및 주문 간 지연은 입력 슬리피지 외에는 모델링하지 않습니다.
- 실제 SOXL 상장 전 합성 이력은 지원하지 않습니다. SOXL 데이터가 없는 날짜는 비교에서 제외하며 전일 가격으로 보간하지 않습니다.
- 문서의 추가 하락구간 “큰수 매수”는 정확한 가격 사다리 공식이 없고 현재 웹앱도 주문을 산출하지 않으므로 임의로 만들지 않았습니다.
- 30분할은 기존 백테스트 호환을 위해 기울기와 리버스 비율을 보간한 실험 모델입니다. 공식 문서와 현재 웹앱이 명시적으로 지원하는 값은 20/40분할입니다.
- 본 도구는 연구용이며 미래 수익이나 실제 체결을 보장하지 않습니다.
